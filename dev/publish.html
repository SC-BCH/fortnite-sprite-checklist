DEV Publish PROMOTION ROUTE hub に戻る
このページは本番反映そのものではなく、draft から publish へ昇格させるための形を固定するための dev 用ガイドです。

1. 前提

  1. editor で座標を調整済みであること。
  2. viewer で同じ画像を使って表示確認済みであること。
  3. verification の確認項目を通過していること。
editor viewer verification

2. publish に上げる対象

  • 画像ファイル
  • version 付き board JSON
  • version 付き events JSON
  • manifest
  • publish record

publish 時は HTML 本体ではなく manifest の参照先を切り替える のが基本方針です。

3. version 付きファイル例

./images/imagessprseireiv3.png ./boards/sprite-seirei.20260626.json ./events/sprite-seirei.20260626.events.json

4. manifest テンプレート

dev/publish-manifest.template.json をベースに本番用 manifest を組み立てます。

{ "boardId": "sprite-seirei", "version": "YYYYMMDD", "image": "./images/imagessprseireivX.png", "board": "./boards/sprite-seirei.YYYYMMDD.json", "events": "./events/sprite-seirei.YYYYMMDD.events.json" }

5. publish record テンプレート

dev/publish-record.template.json を使って、何を確認して publish したかを残します。

{ "boardId": "sprite-seirei", "version": "YYYYMMDD", "verifiedAt": "YYYY-MM-DD", "verifiedBy": "", "verification": { "editorSelectionMatch": true, "editorDragMoveOk": true, "editorResizeOk": true, "viewerClickMatch": true, "viewerPngMatch": true, "completeBadgeOk": true, "itemCountMatch": true } }

6. publish 判定

  • PNG 保存結果がずれない
  • クリック位置とチェック表示がずれない
  • COMPLETE 表示が問題ない
  • 件数が一致している
  • publish record の verification がすべて true

この条件が通らない限り、manifest は切り替えない前提にします。
