JSON editor / viewer を使った後に、このページの順で確認すると「publish へ上げてよいか」の判断を固定できます。

1. 事前条件

  1. editor と viewer で 同じ画像 を選択していること。
  2. dev/boards/sprite-seirei.draft.json の対象件数が想定通りであること。
  3. legacy dev と比較する場合は、旧画面の表示と大きく乖離していないこと。
editor を開く viewer を開く

2. editor 確認

  1. 選択した項目が一覧と画像上の矩形で一致すること。
  2. クリック再配置、ドラッグ移動、四隅リサイズが正しく効くこと。
  3. x / y / w / h を数値で変更すると矩形が即時反映されること。
  4. JSON 出力に同じ値が反映されること。
  5. 画像端で矩形が範囲外に飛ばないこと。

上の 5 点が通れば editor 側は OK 判定。

3. viewer 確認

  1. 同じ画像を選択して、クリック位置とチェック表示がずれていないこと。
  2. 全解除が効くこと。
  3. 名前・文字色・サイズ変更がオーバーレイへ反映されること。
  4. PNG 保存時もチェック位置とオーバーレイ位置がずれていないこと。
  5. 全件チェック時に COMPLETE 表示が期待通り出ること。

上の 5 点が通れば viewer 側は OK 判定。

4. publish 判定

  • publish 可: editor / viewer の両方で問題なし。
  • publish 保留: クリック位置、PNG 出力、COMPLETE 表示のいずれかにズレがある。
  • 差し戻し: 画像自体が違う、または件数が違う。

5. 反映前メモ

publish 前に残すべきメモの例です。

  • 対象画像名
  • 確認した viewer / editor の日付
  • ズレ修正を入れた項目 ID
  • legacy dev と差があった項目
